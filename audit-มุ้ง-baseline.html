A. ชนิดมุ้ง (product / option ที่ระบบมีให้เลือก)

รายการ (ใบจริงเรียก)ใบจริงเขียนอะไรระบบมี + ออกในใบไหมสถานะ + root cause
มุ้งเฟรมเล็ก “มุ้งเฟรมเล็ก ผ้าไฟเบอร์ดำ/เทา” · ใช้บ่อยสุดในใบจริง (บานเปิด/เลื่อน) มี — product imp21/imp22 + addon ต่อบาน imp21/imp22 · ออกในใบ “พร้อมมุ้งเฟรมเล็ก” ✅ ตรง
มุ้งเฟรมใหญ่ “มุ้งเฟรมใหญ่ ผ้าไฟเบอร์ดำ” (บานเลื่อน/สลับ) มี — product imp23 + addon imp23 · ออก “พร้อมมุ้งเฟรมใหญ่” ✅ ตรง
มุ้งจีบ “มุ้งจีบ (เฟรมสี.../ผ้าไฟเบอร์ดำ)” · มีหลายแบบ SD/ตีนตะขาบ มี — product imp28, addon mj_sd_basic / mj_keep_* · ออก “พร้อมมุ้งจีบ” ✅ ตรง
มุ้งจีบนิรภัย “มุ้งจีบนิรภัย” · เปิดได้ทางเดียว ติดตาย 1 ทาง (QC) มี — imp31 (สแตนเลส 0.8มม.) + mj_screen_safety + addon · ออกในใบ ✅ ตรง
มุ้งม้วน “ใส่มุ้งม้วน หรือ มุ้งจีบ” (QC) มี — imp29 (ม้วน) + mj_kick_150/300/600 (ม้วนเตะ) · ออกในใบ ✅ ตรง
มุ้งนิรภัย (ไม่จีบ) ใบจริงแยกชัด: “มุ้งนิรภัย (เฟรมลายไม้สีไวท์โอ๊ค)” / “มุ้งนิรภัย (เฟรมสีเทาซาฮาร่า)” — เป็นมุ้งเฟรมบานแน่น คนละตัวกับ “จีบนิรภัย” ระบบมีแต่ “มุ้งจีบนิรภัย” (imp31/mj_screen_safety) · ไม่มี “มุ้งนิรภัย” แบบเฟรมตายให้เลือกแยก 🟡 เช็คซ้ำ
อาจถือว่า imp31 ครอบคลุม แต่ใบจริงเขียนแยกชื่อ — ยืนยันกับพี่นัทว่าเป็นตัวเดียวกันไหม
มุ้งแม่เหล็ก พับเปิดได้ ไม่พบในใบตัวอย่าง (แต่อยู่ในคลังออปชั่น) มี — imp30 🟡 เช็คซ้ำ
ระบบมี แต่ไม่มีใบจริงยืนยันรูปแบบข้อความ

B. รายละเอียดงานมุ้ง (ผ้า / สีเฟรม / เส้นคาด)

รายการใบจริงเขียนอะไรระบบออกในใบไหมสถานะ + root cause
ผ้ามุ้งไฟเบอร์ ดำ/เทา “ผ้าไฟเบอร์ดำ” (10×) · “ผ้าไฟเบอร์เทา” (5×) — สลับใช้ตามงาน มี dropdown screenFabric/mosqFabric · default render = “ผ้าไฟเบอร์ดำ” · มีตัวเลือกเทาด้วย ✅ ตรง
ผ้ามุ้งกันแมว/หมา “ผ้ามุ้งกันแมว” มี — screenfabric=cat +800/ตร.ม. · ออก “ผ้ากันแมว/หมา” ✅ ตรง
สีเฟรมมุ้ง (เฟรมสีเทาซาฮาร่า / ลายไม้ไวท์โอ๊ค / ไวท์แบมบู KL19022WD / อบขาว) ใบจริงระบุสีเฟรมมุ้งเสมอ เช่น “มุ้งจีบ (เฟรมสีเทาซาฮาร่า/ผ้าไฟเบอร์ดำ)”, “เฟรมลายไม้สีไวท์โอ๊ค”, “เฟรมลายไม้ไวท์แบมบู (KL19022WD)” มี dropdown สีเฟรม (sahara/kl_special/kl_wood — มีผลต่อ ราคา ×0.5) แต่ ข้อความรายละเอียดในใบไม่พิมพ์สีเฟรมออกมา · addon มุ้งออกแค่ “มุ้งจีบ ผ้าไฟเบอร์ดำ” (genQuote line ~3343-3348) · standalone มุ้งออกแค่ผ้า (line ~3170-3178) 🔴 ขาด
root cause: genQuote ไม่ดึง mosqcolor/mscolor มาต่อข้อความ — ราคาคิดถูกแต่ใบไม่บอกลูกค้าว่าเฟรมสีอะไร
เส้นคาดมุ้ง (screen brace) QC บังคับระบุ: เฟรมเล็ก-หน้าต่างเปิด = 1 เส้น · บานเลื่อนทั่วไป = เส้นคู่ · ประตูเลื่อนแบ่ง3ช่อง = 2 เส้น · เฟรมใหญ่ = ไม่ใส่ · นิรภัยสูง>2ม. = คาดสูง 0.90ม. — “ระบุตำแหน่งเส้นคาดมุ้ง + วาดลงแบบ” ระบบไม่มี option เส้นคาดมุ้งเลย · ไม่มีช่องกรอก ไม่ออกในใบ (grep “เส้นคาดมุ้ง” ใน index.html = ไม่พบ) 🔴 ขาด
root cause: ไม่มี product/option — เป็นรายละเอียดผลิต/แบบ ใบเสนอราคาบางใบไม่ลงก็ได้ แต่ QC ต้องการ (เช็คซ้ำว่าต้องลงใบหรือลงแบบ)
ผ้าสแตนกันหนู 0.3มม. มีในระบบ ใบจริงพบ “ผ้าสแตน” มี — imp35 + screenfabric=rat +1,200/ตร.ม. ✅ ตรง
ม่านรังผึ้ง / Blackout / ทูเวย์ / ตีนตะขาบ ไม่พบในใบตัวอย่าง 16 ใบ (ของใหม่) มีครบ — mj_blackout / mj_sd_twoway / mj_keep_twoway / mj_keep_honey ออกในใบ ✅ ระบบเกินใบจริง (ไม่เสีย)

C. ความถูกต้องของการคิดมุ้ง (เชิงราคา/พฤติกรรม)

หัวข้อเกณฑ์ (ใบจริง/QC)ระบบทำอะไรสถานะ
OPTION ลดราคาเมื่อเปลี่ยนเป็นมุ้งเฟรมเล็ก ใบจริงมีบรรทัด “OPTION: หากเปลี่ยนเป็นมุ้งเฟรมเล็ก ราคาลดลง X บาท” หลายใบ ระบบคิดเป็นรายการแยก/addon ตามที่เลือก — ไม่มีกลไกสร้างบรรทัด OPTION เปรียบเทียบ-ส่วนลดอัตโนมัติ 🟡 ต่างวิธี
ไม่ใช่บั๊ก แต่รูปแบบนำเสนอต่างจากใบจริง
มุ้งจีบนิรภัย เปิดทางเดียว QC: เปิดได้ 1 ทาง ติดตาย 1 ทาง ระบบไม่มีหมายเหตุนี้ในใบ (เป็นข้อมูลผลิต) 🟡 เช็คซ้ำ
มุ้งม้วนเตะ เกินความกว้างรุ่น ต้องเปลี่ยนรุ่นใหญ่ขึ้น มี — เตือน “รับกว้าง ≤ X ม. · เปลี่ยนรุ่นมุ้งใหญ่กว่า” + ไม่คิดราคา (กันเก็บเงินขาด) ✅ ดี
มุ้งคิดตามช่องเปิด (บานสลับ = ครึ่ง) พี่นัท: มุ้งจีบ=เต็มช่องเปิด · มุ้งเลื่อน+สลับ=ครึ่งช่องเปิด มี — logic line ~1483-1499 คิดตามช่องเปิด/หารครึ่งบานสลับ ✅ ดี

สรุปสิ่งที่ต้องเพิ่ม/แก้ (จัดลำดับความสำคัญ)

  • P1 (ผิดแน่ — กระทบใบลูกค้า): เพิ่มสีเฟรมมุ้งลงข้อความรายละเอียดในใบ — ปัจจุบันเลือกสีเฟรม (ซาฮาร่า/ไวท์โอ๊ค/ไวท์แบมบู) แล้วคิดราคาถูก แต่ใบพิมพ์แค่ “มุ้งจีบ ผ้าไฟเบอร์ดำ” ไม่บอกสีเฟรม ทั้ง addon (genQuote ~3343-3348) และ standalone (~3170-3178). ใบจริงระบุสีเฟรมทุกครั้ง
  • P2 (น่าสงสัย): เส้นคาดมุ้ง — QC บังคับระบุตำแหน่ง/จำนวนเส้นคาดมุ้ง (1 เส้น / เส้นคู่ / 2 เส้น / ไม่ใส่) ระบบไม่มี option และไม่ออกในใบ. ตัดสินใจกับพี่นัทว่าจะลงใบเสนอราคา หรือลงเฉพาะแบบ (drawing)
  • P2: แยก “มุ้งนิรภัย” (เฟรมตาย) ออกจาก “มุ้งจีบนิรภัย” — ใบจริงเขียนสองชื่อนี้แยกกัน ระบบมีแต่จีบนิรภัย. ยืนยันว่าเป็นตัวเดียวกันหรือควรเพิ่ม product
  • P3 (รูปแบบ): บรรทัด “OPTION: เปลี่ยนเป็นมุ้งเฟรมเล็ก ราคาลด X” — ใบจริงนิยมแสดงทางเลือก/ส่วนลด ระบบยังไม่มีรูปแบบนี้ (เชิงนำเสนอ ไม่ใช่บั๊กราคา)
