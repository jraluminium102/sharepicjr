รายการที่เทียบ

เกณฑ์เทียบ: ใบจริง QT2024080066 (คุณหญิง) — มีงานหลังคา 3 จุด (ไวนิล+ลามิเนต / โพลีตัน+ระแนง / กันสาด) · เสริมด้วยคลัง “เช็คลิสต์ Glass House/หลังคา” จากเอกสาร QC คุณอิงควัชร์ (หน้า 3) · ตรวจ ปีเตอร์/ขวัญ/บุญญารัตน์ แล้ว ไม่มีงานหลังคา จึงไม่ใช้เป็นเกณฑ์.

หัวใจของปัญหา (root cause เดียว ครอบเกือบทุกข้อแดง): ตัวคิดเงินหมวดหลังคา (calc บรรทัด ~1401) บวกราคา “ของเสริมหลังคา” ครบทุกตัว (rfgut/rfbeam/rfchain/rfpvc/rfhs/rfpole/rfeave/rfgc/rfbc) แต่ตัวเขียนข้อความลงใบ (genQuote หมวดหลังคา บรรทัด ~3197–3231) เขียนลงรายละเอียดงานแค่: ชนิด+สีหลังคา / โครง+แป / บรรทัดรางน้ำ (เฉพาะถ้าเลือก “ปลายหลังคา=รางน้ำ”) / เพลทเหล็กล่าง / เหล็กดึง — ของเสริมที่เหลือ ถูกคิดเงินแต่ไม่โผล่ในใบ → ลูกค้าจ่ายของที่มองไม่เห็นในเอกสาร.

ตารางเทียบ (รายการใบจริง ↔ ระบบ)

รายการ (ใบจริง) ใบจริงเขียนอะไร ระบบมี option? ออกลงใบครบ? ขาด/ผิด + root cause
ชนิดหลังคา + สีวัสดุมุง “หลังคาไวนิล สีขาว” / “หลังคาโพลีตัน หนา 3 มม. สีไอซ์ (603S Ice)” มี (ไวนิล/ดีไลท์/โพลีตัน/ลามิเนต + dropdown สี) ครบ ออก “หลังคาไวนิล สีเทา” ได้ถูก. เช็คซ้ำ สีโพลีตันในระบบไม่มี “ไอซ์ 603S” (มี Clear/Opal/Bronze/Green/Grey/Silver…) — รายการสีอาจยังไม่ครบของจริง.
แป (เดี่ยว/คู่) “( แปคู่ )” มี (o-roofbatten: แปเดี่ยว/แปคู่) ครบ ออก “(แปคู่)” ต่อท้ายบรรทัดโครง ตรงใบจริง.
โครงสร้าง “โครงอลูมิเนียม” มี (o-roofframe: อลู / เหล็กชุบซิงค์) ครบ ออก “โครงอลูมิเนียม สี…” / “โครงเหล็กชุบซิงค์” ตรง.
รางน้ำอลูมิเนียม + ความยาว “พร้อมรางน้ำอลูมิเนียม … ปล่อยปลาย” + ใน รายละเอียด: “รางน้ำอลูมิเนียม” มี (o-rfgut: อลู S / สแตน M / สแตน L + ยาว(ม.)) ไม่ขึ้นใบ คิดเงิน(rfgut×len) แต่ไม่เขียนลงใบ. บรรทัด “รางน้ำ (ท่อ สีขาว)” ที่ระบบออก เป็นแค่ป้าย “ระบบรางน้ำ” (cosmetic) ของช่อง rfend=รางน้ำ ไม่ใช่รางน้ำที่คิดเงินจาก o-rfgut. root: genQuote ไม่ดึง rfgut/len.
ตะแกรงกันใบไม้ “+ ตะแกรงกันใบไม้” / “ตะแกรงพลาสติกกันใบไม้ สีดำ” ไม่มี ขาด ไม่มี option “ตะแกรงกันใบไม้” เลย (ทั้ง UI/calc/genQuote). root: ไม่มี product/option. ใบจริงระบุทุกงานที่มีรางน้ำ.
ท่อน้ำทิ้ง PVC + ขนาด/สี “มีท่อน้ำทิ้ง PVC ปล่อยปลาย” / “ท่อน้ำทิ้ง PVC 2.5 นิ้ว สีขาว” มี (o-rfpvc “ครอบท่อ PVC” ยาว(ม.)) ไม่ขึ้นใบ + คำไม่ตรง คิดเงิน(ครอบ PVC) แต่ไม่เขียนลงใบ. อีกทั้งใบจริงเป็น “ท่อน้ำทิ้ง PVC 2.5 นิ้ว” (ตัวท่อ) ส่วนระบบเรียก “ครอบท่อ PVC” (งานครอบ) — คนละสิ่ง อาจสับสน. root: genQuote ไม่ดึง + ไม่มีช่องขนาดท่อ/สี.
วัสดุปิดรอยต่อแผ่นหลังคา “วัสดุปิดรอยต่อแผ่นหลังคา สีขาว” ไม่มี ขาด ไม่มี option. root: ไม่มี. ใบจริงระบุในรายละเอียดงานหลังคาแทบทุกใบ.
ต่อปลายหลังคา (วัสดุผสม) “หลังคาไวนิล (แปคู่) ต่อปลายหลังคากระจกลามิเนต” + “หลังคากระจกลามิเนตใส ฟิล์มเขียว 4+4 มม.” ไม่มี (1 จุด = 1 ชนิดเท่านั้น) ขาด ระบบทำหลังคาผสม 2 วัสดุในจุดเดียวไม่ได้. root: โครงสร้าง product 1 ชนิด/รายการ. ใบจริงผสมไวนิล+ลามิเนตในชุดเดียว.
เสริมเพลทเหล็กล่าง (รับล่าง) “เสริมเพลทเหล็กรับด้านล่าง” มี (o-rfplate checkbox) ครบ ออก “เสริมเพลทเหล็กล่าง (รับล่าง)” ลงใบได้ (เป็น 1 ใน 2 ของเสริมที่ genQuote เขียน). ยืนยันจาก render ตรง.
โซ่รางน้ำ (สี+ลาย) QC: “เพิ่มโซ่รางน้ำมั้ย ระบุสี+ลายโซ่” (ออปชั่นมาตรฐาน) มี 2 ทาง: ป้าย rfend=รางน้ำ→โซ่(ลาย/สี) ไม่คิดเงิน · o-rfchain (3,000/เส้น) คิดเงิน บางส่วน บรรทัด “รางน้ำ (โซ่ลาย… สี…)” ออกได้ เฉพาะเมื่อเลือก rfend=รางน้ำ+ระบบ=โซ่ (cosmetic). แต่ o-rfchain ที่ คิดเงินจริง ไม่ขึ้นใบ. ของซ้อนกัน 2 ที่ ชวนสับสน. root: 2 กลไกแยกกัน.
ปล่อยปลายยื่น / ปิดปลายกันน้ำ “ปล่อยปลาย” (ใบจริง) · QC: “ไม่มีรางน้ำ 1.ปล่อยปลายยื่น 10ซม. 2.โครงปิดปลายกันน้ำ” มี (rfend=ปล่อย/รางน้ำ · o-rfeave “ปิดปลายกันน้ำ” 1,000/ม.) ไม่ขึ้นใบ rfend=ปล่อย → genQuote ไม่เขียนบรรทัดอะไรเลย (ใบจริงเขียน “ปล่อยปลาย” ชัดเจน). o-rfeave คิดเงินแต่ไม่ขึ้นใบ. root: genQuote ข้ามกรณีปล่อย + ไม่ดึง rfeave.
ซ่อนสโลปหลังคาเดิม QC: “งานปิดซ่อนสโลปหลังคาเดิม สมาร์ทบอร์ดทาสี 6มม. / คอมโพสิต 3มม.” มี (o-rfhs: comp 3,500 / smart 4,000 + สูง/ยาว/ด้าน) ไม่ขึ้นใบ คิดเงินครบ (สูตรพื้นที่) แต่ไม่เขียนลงใบ. root: genQuote ไม่ดึง rfhs.
คานเหล็กถัก / เสาแผง / เสา 4"x8" / ครอบคาน / ปิดซ่อนราง QC + ใบจริงระบุเสริมโครง/เสา/ครอบในงานกันสาด-หลังคา มี (o-rfbeam / o-rfpolep / o-rfpole4 / o-rfbcw,bcl / o-rfgcw,gch,gcl) ไม่ขึ้นใบ ทั้งกลุ่มคิดเงินครบ แต่ genQuote ไม่เขียนตัวไหนลงใบเลย. root: เดียวกัน — genQuote เขียนแค่ rfplate/rfpull.
ระแนงใต้หลังคา QC: “เพิ่มระแนงใต้หลังคาไหม มี 2 แบบ 1x5 ซม. / 1"x1.6"” แยกหมวด (มี product ระแนง แต่ไม่ผูกในชุดหลังคา) ต้องแยกรายการ ทำได้แต่ต้องเพิ่มเป็นรายการระแนงต่างหาก ไม่รวมในรายละเอียดหลังคาจุดเดียว (ใบจริงคุณหญิงข้อ 2 ก็แยกระแนง+หลังคาในชุดเดียว). root: ระแนงไม่ใช่ option ของหลังคา. เช็คซ้ำ

ระดับเรื่องเหตุผล
P1 genQuote หมวดหลังคา ต้องเขียน “ของเสริมที่คิดเงิน” ทุกตัวลงรายละเอียดงาน (รางน้ำ+ยาว, ท่อ/ครอบ PVC, ซ่อนสโลป, โซ่, คานเหล็กถัก, เสา, ปิดปลาย, ครอบคาน, ปิดซ่อนราง) ลูกค้าจ่ายเงินแต่ของไม่ขึ้นเอกสาร = เสี่ยงข้อพิพาท/ตกหล่นตอนผลิต. กระทบทุกใบที่มีงานหลังคา.
P1 เพิ่ม option “ตะแกรงกันใบไม้” และ “วัสดุปิดรอยต่อแผ่นหลังคา (สี)” ใบจริงระบุแทบทุกงานหลังคาที่มีรางน้ำ — ตอนนี้ไม่มีให้เลือกเลย.
P2 กรณี rfend=ปล่อย ให้เขียน “ปล่อยปลายยื่น ~10 ซม.” ลงใบ (ตาม QC) แทนที่จะเว้นว่าง ใบจริงเขียน “ปล่อยปลาย” ชัดเจน เป็นข้อมูลการติดตั้ง.
P2 รวมกลไกโซ่/รางน้ำที่ซ้อนกัน 2 ที่ (ป้าย cosmetic rfend กับ o-rfgut/o-rfchain ที่คิดเงิน) ให้เหลือทางเดียว ตอนนี้สับสน: เลือกโซ่ที่หนึ่งฟรี อีกที่คิดเงิน 3,000/เส้น.
P3 รองรับหลังคาผสม 2 วัสดุในจุดเดียว (เช่น ไวนิล + ต่อปลายลามิเนต) มีในใบจริง (คุณหญิง) แต่ไม่บ่อย — workaround แยกรายการได้ชั่วคราว.
P3 ทบทวนรายการ “สีโพลีตัน” ให้มี “ไอซ์ 603S” / สีที่ขายจริง ใบจริงใช้สีไอซ์ ระบบไม่มี. (เช็คซ้ำกับฝ่ายขาย)
