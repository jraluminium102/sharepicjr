1. ชื่อหมวด / label — สื่อความหรือยัง?

ออปชั่น (ปัจจุบัน)ปัญหาเสนอแก้ระดับ
อุปกรณ์เสริม / ออปชั่น
summary ของ <details> หลัก		 "อุปกรณ์เสริม" กำกวม — เซลล์ไม่รู้ว่าคือ สี/กระจก/มือจับ หรือสิ่งที่แตะได้ทุกวัน ไม่สื่อว่า "กรอกที่นี่เพื่อปรับราคา" ⚙ ปรับแต่งรายการนี้
หรือ "รายละเอียด / ออปชั่น"		 P1
🚪 ชนิด / การเปิด
groupGHOpts bucket		 keyword ใน bucket นี้กว้างเกินไป — รับทั้ง ธรณี/ราง/สีอบ/ลาย/โช๊ค/ชนิดชั้น(ตู้) → ของไม่เกี่ยวกับ "ชนิด/การเปิด" หลุดเข้า แยกเป็น 2 หมวด: "🚪 ชนิด / การเปิด" (ชนิดการเปิด/ธรณี/ราง) + "🎨 สี / ผิว" (สีอบ/ลาย/ผิว) · ตัด keyword ที่ซ้อนออก P1
➕ อื่นๆ (ครอบวงกบ / ดรอปพื้น / หมายเหตุ) ชื่อยาวเกินไปสำหรับ summary · "อื่นๆ" ซ่อนของที่ใช้บ่อย (ครอบวงกบ, แผ่นทึบล่าง, รื้อของเดิม) ไว้ด้วยกัน → เซลล์หาไม่เจอ "➕ งานเพิ่มเติม" · ย้ายของที่ใช้บ่อยขึ้นหมวดก่อน P1
📐 คาดตาราง "คาดตาราง" เป็นศัพท์ช่าง — ลูกค้าบางคนอาจไม่รู้ว่าคืออะไร แต่ใช้ใบลูกค้าด้วย → อยู่ในจุดที่ถูกอยู่แล้ว คงชื่อไว้ แต่เพิ่ม sub-label: "📐 คาดตาราง (เส้นลวดลาย)" P2
🔩 ของเสริมหลังคา รับทั้ง เสาแผง/คานเหล็ก/ครอบท่อ/ปิดซ่อนราง/ซ่อนสโลป → หลากหลายเกินไปสำหรับชื่อเดียว "🔩 โครงสร้าง / ของเสริมหลังคา" P2
📦 รายละเอียดตู้ ชื่อดี แต่ผสมทั้ง "ข้อมูลตู้" (ประเภท/ลึก) กับ "วัสดุ" (ชนิดชั้น/ผนัง) ไว้ด้วยกัน คงชื่อไว้ได้ หรือแยกเป็น 📦 ตู้ · 🪞 วัสดุ ถ้ามีของมาก P3

2. อยู่ถูกหมวดไหม? (ของหลุดที่)

ออปชั่นหมวดปัจจุบันปัญหาควรอยู่ที่ระดับ
วัสดุผนังตู้ (อลูทึบ/กระจก) 📦 รายละเอียดตู้ keyword "วัสดุ" หลุดเข้า 🚪 ชนิด/การเปิด ถ้า kw ปัจจุบันรับ "วัสดุ" — ต้องตรวจ bucket ใหม่ก่อน 📦 รายละเอียดตู้ (ถูกแล้ว ถ้า bucket ไม่หลุด) P1
สีกรอบมุ้ง (o-mosqcolor) 🪟 มุ้ง ถูกที่ แต่ใน mosq-wrap กล่องเดียวรวม ผ้ามุ้ง/สีกรอบ/จำนวนบาน → เลื่อนยากบนมือถือ label "สีกรอบมุ้ง" ถูกกลบด้วยชิป ยังอยู่ใน 🪟 มุ้ง แต่ควรเลื่อน สีกรอบ+ผ้ามุ้ง ขึ้นใกล้ "เลือกชนิดมุ้ง" P1
ฝังรางยู (o-uchannel) ไม่มีหมวด — ลอยต่ำ ก่อนหมายเหตุ รางยูเป็นส่วนของ "ชนิด/การเปิด" (กระทบธรณี) แต่ปัจจุบัน keyword ไม่ match → หลุดไปหมวดอื่น 🚪 ชนิด / การเปิด (เพิ่ม kw 'รางยู'/'ฝังราง') P2
รื้อของเดิม (o-removeold) ➕ อื่นๆ (หมวดสุดท้าย) ใช้บ่อย แต่จมอยู่หมวดท้าย เซลล์ต้องจำว่าอยู่ที่ไหน ย้ายขึ้น 🚪 ชนิด / การเปิด หรือสร้างหมวด "🔨 งานรื้อ / เสริม" P2
งานเสริม/วิศวกรรม (o-extrawork) ➕ อื่นๆ ใช้บ่อยในงานพิเศษ แต่จมหมวดสุดท้ายพับอยู่ ยกขึ้นระดับ visible ถ้าว่างแสดง placeholder สั้น / หรือ stick ไว้บน ➕ อื่นๆ P2
ฉนวนกันความร้อน (o-insul) ลอยใน .i-opts ก่อน groupGHOpts จัด → ไม่แน่ใจว่า bucket ไหน keyword "ฉนวน" ไม่ match bucket ใด → หลุด "อื่นๆ" ซึ่งพับ default เพิ่ม kw 'ฉนวน' เข้า 🚪 ชนิด/การเปิด หรือสร้าง "🏠 วัสดุ / ฉนวน" P2
ดรอปพื้น (o-dfm) ➕ อื่นๆ ใช้น้อย — อยู่ท้ายพอรับได้ แต่ชื่อ "ดรอปพื้น" เป็นศัพท์ช่างล้วน คงที่ แต่เปลี่ยนชื่อ: "ดรอปพื้น (ยกเส้นพื้น · ม.)" P3

3. ลำดับ / ไฮรารคี — เรียงถูกไหม?

จุดปัญหาเสนอแก้ระดับ
กล่อง "มุ้ง" — ลำดับภายใน ลำดับปัจจุบัน: เลือกชนิด → จำนวนบาน → (details ขนาด) → ผ้ามุ้ง → สีกรอบ
ปัญหา: ผ้ามุ้ง/สีกรอบ ซ่อนลึก 2 ชั้น (ต้องเปิด mosq-detail-group แล้วเลื่อน)		 เรียง: ชนิดมุ้ง → ผ้ามุ้ง → สีกรอบ → จำนวนบาน → (พับ) ขนาดเอง
ผ้ามุ้ง+สีกรอบขึ้นก่อน — ตัดสินใจหลักก่อนรายละเอียด		 P1
หมวด "⚙ อุปกรณ์เสริม" พับ default สี/กระจก (ย้ายเข้า box บนสุดแล้ว) โอเค แต่ มือจับ/มุ้ง ยังอยู่ในพับ → เซลล์ต้องกด "อุปกรณ์เสริม" ทุกครั้ง ทั้งที่มือจับ/มุ้งใช้บ่อย ขยาย box อัตโนมัติ (open=true) เมื่อสินค้ามี มือจับ หรือ มุ้ง เป็น default กล่องนี้ · หรือโชว์ "🔧 มือจับ" นอกกล่องพับสำหรับบาน P1
combo "บานกระทุ้งเข้าใน + มุ้งจีบ" ปัจจุบันลอยใน .i-opts ก่อนกลุ่มมือจับ — keyword ไม่ match bucket ใดชัด → หลุด "อื่นๆ" ทั้งที่มีผลต่อราคา เพิ่ม kw 'combo' เข้า 🚪 ชนิด/การเปิด หรือ 🔧 มือจับ/ล็อค P2
สีวัสดุมุง (หลังคา) ลำดับ โชว์ก่อน แต่ละรุ่นมี dropdown ต่างกัน: สีเมทัลชีท / สีชินโคไลท์ / สีวัสดุมุง → label ไม่ consistent (ตัวนำต่าง) ใช้ label เดียว "สี" นำทุกรุ่น (ข้อมูลรุ่นอยู่ใน dropdown แล้ว) ลด cognitive load P2
ม่านซิป — กล่อง gh-box ลำดับ กลุ่มลูกค้า (×retail/project) โผล่ก่อน รุ่น/ผ้า → คนกรอกต้องรู้ multiplier ก่อนรู้สินค้า เรียงใหม่: รุ่น → ผ้า → ระบบควบคุม → กลุ่มลูกค้า
(กลุ่มลูกค้าเป็นข้อมูลสุดท้ายที่ปรับ)		 P3

4. อื่นๆ — ช่องซ้ำ / label สับสน / UX

จุดปัญหาเสนอแก้ระดับ
หมวด-bucket ใน groupGHOpts
🚪 ชนิด/การเปิด keyword		 keyword list ยาว 22 คำ ปนกัน — 'สี ','ลาย','ความลึก','ราวจับ' อยู่ใน bucket "ชนิด/การเปิด" → ของไม่เกี่ยวกับการเปิดหลุดเข้า แยก bucket เป็น 🚪 ชนิด/การเปิด + 🎨 สี/ผิว ใหม่ (ดู tab IA) P1
[Up3] โช๊ค (combo สแตน) ชื่อมี "[Up3]" prefix (dev tag) หลุดโชว์ให้เซลล์เห็น — ศัพท์ dev ไม่ควรอยู่ใน label ผู้ใช้ ลบ [Up3] ออก: "โช๊ค (combo สแตน)" P2
ค่าประตู (เลือกรุ่น) vs ราคาประตู (กรอกเอง) สองช่องติดกันแต่ทำงานแยกกัน — เซลล์อาจกรอกทั้งคู่โดยไม่รู้ว่ากรอก >0 ช่องหลังจะ override ช่องแรก รวมเป็น 1 ช่อง + logic: เลือกรุ่น → โชว์ราคา auto → link "แก้ราคา" (pattern เดิมของมอเตอร์หลังคา) P2
สีเฟรมอลูมิเนียม (frameless_door) บานเปลือย (frameless) มีช่อง "สีเฟรม" ทั้งที่ชื่อสินค้าบอกว่า "ไม่มีเฟรม" — อาจสับสน เปลี่ยนเป็น "สีขอบ/trim อลูมิเนียม" หรือ "สีฟิตติ้ง" P2
มีคาดตาราง (คิดตามสูตร) วงเล็บ "(คิดตามสูตร)" ไม่จำเป็น — เซลล์รู้ว่าระบบคิดราคาเอง · ยาวเกิน "มีคาดตาราง" (ตัดวงเล็บ) · ราคา show ใน preview อยู่แล้ว P3
ช่องแสง (กระจก) พื้นที่ ... ตร.ม. label ยาว 2 ชั้น — "ช่องแสง (กระจก)" + unit ซ้ำใน placeholder "ช่องแสงกระจก (ตร.ม.)" · ตัด "เว้น 0 = ทึบล้วน" เป็น hint icon แทน P3
คงเดิม ย้าย ย้ายหมวด ชื่อ เปลี่ยนชื่อ เพิ่ม bucket ใหม่ ตัด รวมหรือตัด พับ พับ default

  2. เปลี่ยนชื่อ summary หลัก "อุปกรณ์เสริม / ออปชั่น" → "⚙ ปรับแต่งรายการนี้"
    คนกรอกไม่รู้ว่าต้องกดตรงนี้เพื่อใส่มือจับ/มุ้ง — ชื่อใหม่บอก action ชัดกว่า
    แก้ใน addItem template: summary ของ .optbox
  3. เปิด .optbox อัตโนมัติสำหรับสินค้าที่มีมือจับ/มุ้ง
    ตอนนี้ ob.open ขึ้นกับ p.ranae|p.cabinet|p.gate ฯลฯ แต่ไม่รวม p.mosquito|p.digihandle → มุ้ง/มือจับซ่อนพับ
    แก้บรรทัด ob.open: เพิ่ม ||p.mosquito||(_cmechOK||_xOK||_awnOK||p.digihandle)
  4. ปรับลำดับภายใน กล่องมุ้ง: ผ้ามุ้ง+สีกรอบ ขึ้นก่อนจำนวนบาน
    ตัดสินใจสำคัญ (ชนิด → ผ้า → สี) ควรอยู่บนสุด · รายละเอียด (จำนวน/ขนาด) ตามหลัง
    แก้ใน buildItemOpts ส่วน p.mosquito: ย้าย h+=(ผ้า/สีกรอบ) ขึ้นก่อน mosq-panels-bar
  5. ลบ "[Up3]" prefix ออกจาก label โช๊ค
    dev tag หลุดโชว์ให้เซลล์เห็น — ใช้เวลาแก้ไม่ถึง 1 นาที
    แก้บรรทัด p.closer: เปลี่ยน label '[Up3] โช๊ค' → 'โช๊ค'

ต้องถามพี่นัท — ยังตัดสินไม่ได้

  • bucket "🎨 สี / ผิว" ควร open หรือ พับ default?
    ถ้า open → ฟอร์มยาวขึ้น · ถ้าพับ → เซลล์ต้องกดเพิ่ม 1 ครั้ง · สีเลือกบ่อยแค่ไหนต่องาน?
  • "รื้อของเดิม" และ "งานเสริม/วิศวกรรม" ควรอยู่นอกกลุ่มพับหรือยัง?
    ถ้าพี่นัทตอบว่าใช้บ่อยทุกออเดอร์ → ย้ายขึ้นมาโชว์ · ถ้าใช้น้อย → คงพับไว้ได้
  • label "ดรอปพื้น" ลูกค้าใช้คำนี้ด้วยหรือเปล่า?
    ถ้าลูกค้ารู้จัก → คงไว้ · ถ้าเป็นศัพท์ช่างล้วน → เปลี่ยนเป็น "ยกเส้นพื้น (ม.)" หรือ "เว้นพื้น (ม.)"
  • bucket "🎨 สี / ผิว" ควรรวมกับ 🚪 ไหม สำหรับบางสินค้า?
    เช่น บานเปิด — สีอบแทบไม่มี (สีตามอลู) ทำให้ bucket นี้ว่างเปล่า → อาจไม่ต้องสร้างถ้าว่างบ่อย
  • "ผ้ามุ้ง" ควรขึ้นก่อน "จำนวนบานมุ้ง" จริงไหม?
    เซลล์ตัดสินใจเรื่องผ้าก่อนหรือเรื่องจำนวนบานก่อน? (ถ้าราคาผ้าไม่ขึ้นกับบาน → ผ้าขึ้นก่อนสมเหตุผล)

จุดที่ตัดสินได้เลย (ไม่ต้องถาม)

  • ลบ [Up3] ออกจาก label โช๊ค — ชัดเจนว่าเป็น dev tag หลุด
  • แยก bucket "🎨 สี/ผิว" ออกจาก 🚪 — keyword ปัจจุบันกว้างเกินไปทุก dev เห็นด้วย
  • เปลี่ยนชื่อ summary หลัก "อุปกรณ์เสริม" → "⚙ ปรับแต่งรายการ" — ไม่กระทบ logic
  • ตัดวงเล็บ "(คิดตามสูตร)" ออกจาก "มีคาดตาราง" — ฟุ่มเฟือย
  • ลำดับหลังคา สีวัสดุมุง → ใช้ "สี" label นำ — consistent ทุกรุ่น
