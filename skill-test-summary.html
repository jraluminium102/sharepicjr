🔎 อ่านผลยังไง

ข่าวดี: สกิลทำงานครบ flow ทุก run (render → อ่านใบจริง → เทียบ → PDF → self-check → READ-ONLY) และ เจอบั๊กจริงตรงกันทั้ง with/baseline = ผลน่าเชื่อถือ
• มุ้ง: สีเฟรม/มุ้งนิรภัยเฟรมตัน ไม่ลงใบ
• หลังคา: ของเสริม (เสา/คาน/ซ่อนสโลป/โซ่) คิดเงินแต่ไม่พิมพ์ลงใบ ← บั๊กสำคัญ ลูกค้าจ่ายของที่มองไม่เห็น

ข้อค้นพบสำคัญ: baseline (ไม่ใช้สกิล) ก็ทำได้ดีเกือบเท่า — เพราะโปรเจกต์มี CLAUDE.md + harness + เอกสารชัดมาก · คุณค่าจริงของสกิล = ความสม่ำเสมอ (READ-ONLY เสมอ, self-check เสมอ, รู้ path ไม่ต้องค้น, ฟอร์แมต PDF เดิม) + ไม่ต้องพิมพ์อธิบายยาวทุกครั้ง
