HANDOFF — ราคาบานเปิดยูโร (casement_euro) คิด "ต่อบาน"

ส่งแชทที่ดูแล G1-บานเปิดโดยเฉพาะ · มติพี่นัท 11 มิ.ย. 2026 · เปลี่ยนจาก bucket เดิม → คิดต่อบาน clamp · READ-ONLY (ผู้ตรวจไม่แตะโค้ด)
สรุปสั้น: บานเปิดยูโรเลิกคิด bucket+min18,000+addon เดิม → คิด ราคาต่อบาน แบบ Linear ตามพื้นที่บาน (พื้น 15,000 · วิ่งขึ้นตามเรท ไม่มีเพดาน) แล้ว ×จำนวนบาน · ออปชั่นเสริมทุกอย่างคิดเหมือนเดิม · เคาะเรตแล้ว 11 มิ.ย.

1) สูตร (ราคาฐานบาน) — ✅ เคาะแล้ว: Model B Linear

ราคา/บาน = 15,000 + max(0, พื้นที่บาน − 1.2) × 4,166.67  (พื้น 15,000 · ไม่มีเพดานบน)
ราคารวม = ราคา/บาน × จำนวนบาน · พื้นที่บาน = กว้าง×สูง ที่กรอก (= ต่อ 1 บาน)

ลากเส้น 2 จุด: 1.2 ตร.ม. → 15,000  ถึง  3.6 ตร.ม. → 25,000 · ความชัน 4,167 บาท/ตร.ม.
แนะนำเก็บเป็นตัวแปรเดียว ปรับง่าย: const CE={floor:15000, refA:1.2, refP:15000, slope:4166.67};
pricePerPanel = CE.floor + Math.max(0, area - CE.refA) * CE.slope;

3) จุดแก้ในโค้ด (index.html)

  • ยกเลิก (เฉพาะ casement_euro L578): method:'bucket' + min:18000 + addon:{kind:'door',extra:5000} → แทนด้วยสูตร Linear ต่อบานข้างบน (15000 + max(0,area-1.2)×4166.67)
  • ราคารวม = ราคา/บาน(พื้นที่ = w×h ต่อบาน) × panels
  • ปลดจำนวนบาน: L2655 pn.max=2 → 4 (หรือมากกว่า)
  • label ช่องกว้าง: → "กว้าง/บาน (ม.)" ให้ชัดว่ากรอกต่อบาน
  • ลิมิตขนาด (ใหม่): ถ้า w>1.5 หรือ h>3.0 → บล็อก/เตือน "บานเปิดยูโรเกินขนาด (กว้าง≤1.5 สูง≤3.0)" · ไม่มีเพดานราคา (วิ่งตามเรทถึง 4.5 ตร.ม. = 28,750)

4) ออปชั่นเสริม — คิดเหมือนเดิม (สำคัญ)

สูตร clamp/Linear ใช้กับ ราคาฐานบาน เท่านั้น · ออปชั่นพวกนี้ยัง บวกเพิ่มจากราคาฐานตามเดิม ทุกตัว:
  • มือจับดิจิตอล / JR Prime / โช๊ค +5,000 (บานหลัก)
  • มุ้ง · สีอลู (ส่วนต่างสี) · กระจก (glassUp) · ครอบวงกบ · คาดตาราง
  • combo บานกระทุ้งเข้าใน + มุ้งติดตาย
เอาออกอย่างเดียว: addon door +5,000 (รวมในสูตรต่อบานแล้ว)

5) ขอบเขต + เช็คซ้ำ
